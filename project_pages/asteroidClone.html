The Brief

The goal of this application was to explore the concept of applying physical properties such as gravity to an animation. By implementing simple physics functions and applying them to elements within the game, a basic physics engine was created. This allowed objects within the game to respond more realistically. For example, the ship continues to move in any direction once started. This is like the effect that a lack of friction in space causes. Applying these force responses to actions creates for a more enjoyable playing experience as it adds to the illusions built by the game. This project was one part of four that served as the midterm submission for CM2030 Graphics Programming.

The experience

The users goal in the Asteroids clone is to survive as long as possible. The user controls a ship and tries to destroy all the falling asteroids before they hit the planet. The user controls the ship by pressing the directional arrow keys and fires the weapon systems by pressing spacebar. If the ship touches an asteroid or the planet surface, the game ends and must be restarted.

Building Blocks

The application was built using the p5.js JavaScript library, basic HTML code and CSS. There were four main components essential for this game to function, a system of collision detection, bullet, and asteroid management, and finally a system for managing the ship. Each was established as its own class element to keep the code clean and modular. The asteroid game elements were spawned randomly and propelled towards the planet by applying a small gravity force to add acceleration. When a users bullet collides with an asteroid, or leave the screen the bullet is removed from the array that stores its information this also happens for the asteroid. This system allows the game to remain relatively small and efficient.

User Interaction

The user interacts with the experience by controlling the ships direction with the arrow keys and firing its gun system to destroy asteroids with the spacebar

Conclusion

This application was effective at showcasing the value of applying physical forces to game elements and animations. It made for compelling game play by adding interesting challenges, despite the simplicity of the game. This project was the second of four worked on as the midterm submission for the mandatory course Graphics Programming.
