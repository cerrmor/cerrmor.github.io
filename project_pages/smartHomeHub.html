The Brief

The Smart Home Hub served as the midterm project for CM2040 Databases Networks and the Web, aswell as an example of full stack development. The purpose of this project was to build a dynamic web application that included:

  • Database access
  • Procedurally generated web pages
  • Allowed user to visualise, edit, and interact with the data in the database.

The form the project took, was a dashboard for networking and controlling a users Internet of Things (IoT) devices, also known as smart home devices. The project focused on building the different layers required such as, the database SCHEMA or design, the server and the interactive dashboard. This put including real-time control of IoT devices outside the scope of the project.

The experience

The general experience of this application is similer to browsing a webpage. Users access the application through a internet browser of their choice by entering the address of the host server. This takes the user to the applications homepage. From here the user can interact with the application by clicking buttons to move between pages, and filling out form fields to update stored data. As this is a simulation, or proof of concept focused on building the different layers in a full stack development, there is currently no way to network a real IoT device to the system. There is also no user accounts which would be required personalise and protect an individuals smart devices.

Building Blocks

Node.js and Express was used to build a simple web server, a MySQL relational database was used to store the data to be displayed, and the webpages were populated with the required information upon page request.

Navigating the experience

When a user first navigates to the application, they are greeted by the following home page.

The home pages includes a basic greating to welcome the user, it also includes a statement indicating that a new users first action should be to click on the "add new device" button. If the user does this, the application transisitions to the add device page, populating the list of available devices from the database procedurally.

This page allows the user to select a device to add; from a list of twenty available devices. If the user clicks ok without selecting a device, an alert is shown indicating that they have not yet selected anything. Once the user has selected a device and pressed ok, a new page is shown allowing the user to select the devices initial settings. Upon submission, a message indicating the device has been successfully added is displayed and the user is redirected back to the initial add device page.

From here if the user is done adding new devices, they have the ability to view the ones they have already added, by clicking The "device status button". This takes them to a page that lists all user devices and allows viewing editing current settings.

If the user chooses to update a device the following two pages will be shown upon compleating the action.

If the user chooses to delete a device, the user is alerted to the fact that deleting cannot be undone.

After the user has selected ok they are presented with the choice to continue with the deletion or to cancel the action. If they select to cancel, a message indicating the device still exsists is shown. If they choose to proceed, a new message indicating the device was successfully removed is show and the user is redirected back to the device status page.

The last page available to the user is the about page. This page displays the purpose of the application, and introduces the designer.

The actions used to allow the user to navigate the experience were, button clicks, drop down menus, and form fields. Some graphical feed back was also used to indicate the difference between a safe action and a destructive action. For example as seen in the following image, buttons would higlight green if the action was safe and red if it was not.

Navigation of the experience was kept fairly simple and stuck with actions that most users have previous experience using. This allowed for the fastest learning curve with new users.
